🚧 El Problema del Entorno Local

El problema

Has construido un Agente RAG con FastAPI increíble en tu computador. Funciona perfecto en localhost:8000.

💻 Funciona aquí

Tú, el desarrollador, puedes hacer peticiones a la API sin problema.

🚫 Falla allá

Tu cliente, profesor o un servicio externo (como un webhook o un frontend web desplegado) no puede acceder a tu localhost.

La Solución Tradicional: Desplegar toda la arquitectura (Postgres, pgvector, Ollama, FastAPI) en AWS, Azure o GCP. Tarea lenta, compleja y costosa.

🌐 ¿Qué es Ngrok?

Conociendo Ngrok

Ngrok es una herramienta que expone servicios de red locales detrás de firewalls y NATs hacia Internet a través de túneles seguros.

🔌 Instante

Genera una URL pública en segundos sin configurar puertos del router.

🔒 Seguro

Túneles encriptados (HTTPS) automáticos sin manejar certificados SSL.

🔬 Inspección

Permite capturar, inspeccionar y re-ejecutar el tráfico HTTP.

¿Cómo funciona?

🌍 Internet (Usuarios)
→
☁️ Servidores Ngrok
→
🔒 Firewall / Router
→
🤖 Agente (localhost)

1. Ngrok Cloud asigna una URL (ej. https://tu-api.ngrok-free.app).
2. El cliente Ngrok en tu PC inicia una conexión saliente hacia la nube.
3. Las peticiones públicas entran por la nube y bajan por el túnel hacia tu API.

🔑 Clave: Como la conexión es de "adentro hacia afuera", bypassa firewalls sin tener que abrir puertos (Port Forwarding).

🐳 Integración en Docker

Arquitectura en el Proyecto

Hemos "dockerizado" la API y añadido Ngrok al ecosistema.

🌍 Internet
↔
Docker Compose Network
🚇 Ngrok
:4040
↔
🌐 FastAPI
:8000
↔
🐘 Postgres + 🦙 Ollama

El Archivo Compose

# compose.yml
services:
  # ... (Postgres y Ollama)

  api:
    build: .
    container_name: rag-api
    ports:
      - "8000:8000"

  ngrok:
    image: ngrok/ngrok:latest
    container_name: rag-ngrok
    ports:
      - "4040:4040"
    environment:
      - NGROK_AUTHTOKEN=${NGROK_AUTHTOKEN}
    command:
      - "http"
      - "api:8000" # Apunta directamente al contenedor de la API

¿Qué logramos con esto?

1. Cero Instalaciones

No necesitas instalar Ngrok en tu Mac/PC. Todo corre como un contenedor más en la red virtual.

2. DNS Interno Docker

En lugar de apuntar a localhost:8000, el comando es http api:8000, usando la resolución de nombres nativa de Docker.

3. Seguridad del Token

El Authtoken se inyecta de forma segura a través del archivo .env sin exponerlo en el código.

4. Un Solo Comando

docker compose up -d --build despliega todo: base de datos, LLM, API y el túnel al exterior.

🚀 Puesta en Marcha

Paso a Paso

  1. Registrarse en Ngrok: Obtener el token en dashboard.ngrok.com
  2. Configurar entorno: Añadir en .env: 
    NGROK_AUTHTOKEN=tu-token-super-secreto-aqui
  3. Levantar todo: 
    docker compose up -d --build
  4. Disfrutar:

    Revisar en consola la URL generada: docker compose logs ngrok

Dashboard Local (Inspección)

Ngrok provee un panel de control web en el puerto local 4040.

📍 Visita: http://localhost:4040
  • Muestra la URL pública asignada.
  • Muestra un historial de todas las peticiones (headers, body, status).
  • Permite realizar "Replay" de una petición con un clic (ideal para testear la API de LangGraph sin reconstruir la petición).

📝 Conclusiones

Resumen

  • Ngrok es un puente instantáneo entre el exterior (Internet) y el interior (Docker localhost).
  • No reemplaza un despliegue de producción definitivo, pero es vital para la etapa de desarrollo y validación.
  • Gracias a la dockerización de FastAPI, el despliegue es inmutable y escalable en tu máquina.
  • Cualquier persona puede ahora ver el Swagger UI de la API accediendo al endpoint /docs con tu enlace temporal HTTPS.
