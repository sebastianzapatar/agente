¿Qué es LangGraph?

LangGraph es un framework de orquestación de agentes desarrollado por el equipo de LangChain para construir aplicaciones LLM como grafos de estado.

💡 En una frase: LangGraph te permite definir el flujo de tu agente como un grafo dirigido donde cada nodo es un paso del procesamiento y el estado fluye entre ellos.

¿Cuándo usar LangGraph?

✅ Usa LangGraph cuando...

• Necesitas flujos con ciclos o loops
• Construyes agentes multi-paso
• Requieres estado explícito y tipado
• Necesitas human-in-the-loop
• Quieres depurar cada paso
• Vas a producción

⚡ Quédate con LangChain si...

• Tu flujo es lineal (A → B → C)
• Estás prototipando rápido
• Solo necesitas un chatbot simple
• No requieres estado complejo
• Quieres máxima velocidad de desarrollo

Ventajas de LangGraph

🔄 Ciclos

Permite loops y reintentos que LangChain no soporta nativamente.

📊 Estado Tipado

TypedDict define exactamente qué datos fluyen por el grafo.

🔍 Depuración

Inspecciona el estado en cada nodo del pipeline.

🧩 Modular

Cada nodo es independiente, fácil de testear y reutilizar.

👤 Human-in-loop

Pausa el grafo para aprobación humana antes de continuar.

🚀 Producción

Diseñado para ser robusto, escalable y confiable.

🧱 Conceptos Clave

1. Estado (State)

El estado es un diccionario tipado que contiene toda la información que fluye por el grafo.

# Se define con TypedDict para tener tipado estricto
class AgentState(TypedDict):
    input: str                    # Pregunta del usuario
    chat_history: list[BaseMessage]  # Memoria
    standalone_question: str      # Pregunta reformulada
    context: str                  # Documentos recuperados
    response: str                 # Respuesta final
🎯 Clave: Cada nodo recibe el estado completo, lo procesa, y retorna SOLO las claves que quiere actualizar.

2. Nodos (Nodes)

Los nodos son funciones de Python que procesan el estado. Cada nodo hace UNA cosa.

# Un nodo recibe el estado y retorna actualizaciones
def recuperar_documentos(state: AgentState) -> dict:
    """Busca documentos relevantes en pgvector."""
    question = state["standalone_question"]
    docs = retriever.invoke(question)
    context = formatear_docs(docs)
    return {"context": context}  # Solo actualiza 'context'
  • Entrada: El estado global completo
  • Salida: Diccionario con las claves a actualizar

3. Aristas (Edges)

Las aristas definen las conexiones entre nodos — el flujo del grafo.

# Construir el grafo
graph = StateGraph(AgentState)

# Agregar nodos
graph.add_node("reformular", reformular_pregunta)
graph.add_node("recuperar", recuperar_documentos)
graph.add_node("responder", generar_respuesta)

# Definir aristas (conexiones)
graph.add_edge(START, "reformular")
graph.add_edge("reformular", "recuperar")
graph.add_edge("recuperar", "responder")
graph.add_edge("responder", END)

# Compilar
app = graph.compile()

4. Aristas Condicionales

Las aristas condicionales son una de las características más poderosas de LangGraph. Permiten que el flujo tome caminos diferentes según el estado.

# Función que decide el próximo nodo
def decidir_ruta(state: AgentState) -> str:
    """Evalúa el estado y decide a dónde ir."""
    if state["context"] == "No se encontró información":
        return "sin_info"     # → Nodo de respuesta sin info
    return "responder"        # → Nodo de respuesta normal

# Agregar arista condicional al grafo
graph.add_conditional_edges(
    "recuperar",              # Nodo de origen
    decidir_ruta,             # Función de decisión
    {                         # Mapa de rutas
        "sin_info": "disculpar",
        "responder": "responder"
    }
)
🔍 Recuperar
→
🔀 Decisión
→
💬 Responder
🚫 Disculpar

💡 Esto permite branching, loops de reintento, validación y human-in-the-loop.

🔄 Reformular

Si hay historial, reformula la pregunta para que sea autocontenida. Sin historial, la pasa directo.

💻 Código: agent_graph.py

Definición del Estado

from typing import TypedDict
from langchain_core.messages import BaseMessage

class AgentState(TypedDict):
    """Estado global del agente — fluye entre todos los nodos."""
    input: str                       # Pregunta original
    chat_history: list[BaseMessage]  # Memoria de la sesión
    standalone_question: str         # Pregunta independiente
    context: str                     # Docs recuperados
    response: str                    # Respuesta final

El estado es el contrato entre todos los nodos. Cada nodo sabe exactamente qué datos esperar y qué datos producir.

Nodo: Reformular Pregunta

def reformular_pregunta(state: AgentState) -> dict:
    chat_history = state.get("chat_history", [])
    user_input = state["input"]

    # Optimización: sin historial, no reformular
    if not chat_history:
        return {"standalone_question": user_input}

    # Con historial: usar LLM para reformular
    llm = _create_llm()
    prompt = ChatPromptTemplate.from_messages([
        ("system", CONTEXTUALIZE_PROMPT),
        MessagesPlaceholder("chat_history"),
        ("human", "{input}")
    ])
    chain = prompt | llm | StrOutputParser()
    standalone = chain.invoke({
        "input": user_input,
        "chat_history": chat_history
    })
    return {"standalone_question": standalone}

Nodo: Recuperar Documentos

def recuperar_documentos(state: AgentState) -> dict:
    retriever = _create_retriever()
    question = state["standalone_question"]

    # Búsqueda por similitud en pgvector
    docs = retriever.invoke(question)

    if not docs:
        context = "No se encontró información relevante."
    else:
        formatted = []
        for i, doc in enumerate(docs, 1):
            source = doc.metadata.get("source", "?")
            source = source.split("/")[-1]
            formatted.append(
                f"[Doc {i} — {source}]\n{doc.page_content}"
            )
        context = "\n\n".join(formatted)

    return {"context": context}

Nodo: Generar Respuesta

def generar_respuesta(state: AgentState) -> dict:
    llm = _create_llm()
    prompt = ChatPromptTemplate.from_messages([
        ("system", SYSTEM_PROMPT),  # Incluye {context}
        MessagesPlaceholder("chat_history"),
        ("human", "{input}")
    ])
    chain = prompt | llm | StrOutputParser()

    response = chain.invoke({
        "input": state["input"],
        "context": state["context"],
        "chat_history": state.get("chat_history", [])
    })
    return {"response": response}

El prompt del sistema instruye al LLM a responder SOLO con información del contexto proporcionado.

Compilación del Grafo

def build_graph():
    # 1. Crear el grafo con el tipo de estado
    graph = StateGraph(AgentState)

    # 2. Agregar nodos (funciones)
    graph.add_node("reformular", reformular_pregunta)
    graph.add_node("recuperar", recuperar_documentos)
    graph.add_node("responder", generar_respuesta)

    # 3. Definir aristas (flujo)
    graph.add_edge(START, "reformular")
    graph.add_edge("reformular", "recuperar")
    graph.add_edge("recuperar", "responder")
    graph.add_edge("responder", END)

    # 4. Compilar y retornar
    return graph.compile()

Función de Chat con Memoria

def chat(question: str, session_id: str = "default") -> str:
    graph = get_graph()
    history = get_session_history(session_id)

    # Ejecutar el grafo completo
    result = graph.invoke({
        "input": question,
        "chat_history": history,
        "standalone_question": "",
        "context": "",
        "response": ""
    })

    # Guardar en memoria
    history.append(HumanMessage(content=question))
    history.append(AIMessage(content=result["response"]))

    return result["response"]

API FastAPI — Endpoint de Chat

@app.post("/chat", response_model=ChatResponse)
async def chat_endpoint(request: ChatRequest):
    """Ejecuta el grafo LangGraph completo."""
    try:
        respuesta = agent_chat(
            request.pregunta,
            request.session_id
        )
    except Exception as e:
        raise HTTPException(
            status_code=500,
            detail=f"Error: {str(e)}"
        )
    return ChatResponse(
        respuesta=respuesta,
        session_id=request.session_id
    )

📝 Conclusiones

Resumen

  • LangGraph orquesta agentes como grafos de estado dirigidos
  • El estado tipado (TypedDict) hace el flujo predecible
  • Los nodos son funciones puras que procesan el estado
  • Las aristas definen el flujo entre nodos
  • Se combina con LangChain para los bloques individuales
  • Ideal para producción: depurable, extensible, robusto
🎯 Regla de oro: Usa LangChain para construir los bloques, LangGraph para orquestar el flujo.
